T1.合并书本（book）

题目描述：

有 nnn 种书，第 iii 种书有 cic_ici​ 本，读完一本种类为 iii 的书要花费的时间为 tit_iti​ 获得的收益为 wiw_iwi​。
给定 mmm，询问读书时间不超过 mmm 的最大收益是多少。
1≤n≤801 \le n \le 801≤n≤80，1≤m≤1091 \le m \le 10^91≤m≤109，1≤ti≤801 \le t_i \le 801≤ti​≤80，1≤vi,wi≤1091 \le v_i,w_i \le 10^91≤vi​,wi​≤109

题目分析：

关于背包问题一个很假的贪心就是直接按性价比排序，然后从大到小选。

这样直接做不正确的原因就是我们可能按性价比贪心地选填不满背包，而可以丢弃一部分已经选择的物品，换取一些性价比不那么优但是却可以填满整个背包，这样我们的花费变多了，虽然性价比可能下降了但是总体价值变高了。

可以发现本题中 ttt 十分小，也就是说我们如果直接贪心那么只需要在这个的基础上微调一下即可。

如果我们的调整幅度大于 ttt 的最大值很多，那么我们显然将调整来的一部分直接换成等价的性价比更高的物品得到的收益会更高。

对于调整其实就相当于一个背包容量为 O(nt)O(nt)O(nt)，物品种类数为 O(n)O(n)O(n) 的多重背包，因为我们可以选择已选的物品丢弃，所以可以令这种情况下 t,wt,wt,w 为其正常情况下的相反数即可。

可以直接使用二进制优化，复杂度为 O(n2tlog⁡V)O(n^2t\log V)O(n2tlogV)

T2.桂花树（Tree）

题目描述：

给定 nnn，要求求出所有 nnn 个点的弱联通有向图中有向路径的条数总和，答案对给定的模数 modmodmod 取模。

弱联通有向图：若对于一张有向图，将其所有的边都改为双向边之后，其形成一棵树，则称其为一个弱联通有向图。

有向路径：对于一个序列 a1,a2,a3,⋯ ,ama_1,a_2,a_3,\cdots,a_ma1​,a2​,a3​,⋯,am​，存在有向边 (a1,a2)(a_1,a_2)(a1​,a2​)(a2,a3)(a_2,a_3)(a2​,a3​),⋯ ,,\cdots,,⋯,(am−1,am)(a_{m-1},a_m)(am−1​,am​)，则称其为一条有向路径。

1≤n≤1071 \le n \le 10^71≤n≤107，保证 modmodmod 为质数，且满足 n<modn < modn<mod。

题目分析：

一个想法就是枚举路径长度 kkk，然后寻找这个长度的路径出现了多少次。

我们不妨考虑把这一条长度为 kkk 的链拿出来，将其余 n−kn-kn−k 个点挂到这 kkk 个点下面。

问题就转化为了 nnn 个点，构成包含 kkk 棵树的森林，要求给定的 kkk 个点必须在不同的树中的方案数。

根据广义 Cayley 定理，这个的方案数就是 k×nn−k−1k \times n^{n-k-1}k×nn−k−1，但是这是无向树的方案数。

因为我们对于每一棵树都有一个钦定的根，即长度为 kkk 的链上的点，所以我们将其转化为有根树之后，方案数显然依旧为 k×nn−k−1k \times n^{n-k-1}k×nn−k−1。

下面就是考虑对边定向，显然贡献为 2(n−1)−(k−1)2^{(n-1) - (k-1)}2(n−1)−(k−1)。

因为我们一个长度为 kkk 的链必然边被定向好了，所以其不同的排列顺序是会造成不同的答案，所以贡献为 k!k!k!。

因为这 kkk 个点可以任意选取，所以贡献为 (nk)\binom{n}{k}(kn​)。

总结一下，长度为 kkk 的链的贡献其实就是 k×nn−k−1×2(n−1)−(k−1)×k!×(nk)k \times n^{n-k-1} \times 2^{(n-1) - (k-1)} \times k! \times \binom{n}{k}k×nn−k−1×2(n−1)−(k−1)×k!×(kn​)。

注意特判，当 k=nk = nk=n 时 nn−k−1n^{n-k-1}nn−k−1 就没有值了，此时显然贡献为 n!n!n!。

可以通过预处理阶乘、222 的幂和 nnn 的幂，做到 O(n)O(n)O(n) 的复杂度。

T4.Through Another Maze Darkly

题目描述：

黑暗迷宫是一个树形结构，有 nnn 个房间和 n−1n - 1n−1 个走廊，房间编号 1,2,⋯ ,n1, 2, \cdots, n1,2,⋯,n。

黑暗迷宫里面漆黑一片，你看不见自己在哪里。为了辨别方向，每个房间有一个激光指示器，初始指向连接这个房间的某一个走廊。你重复执行如下策略行动：

  • 将当前房间的激光指示器按顺时针方向旋转到下一个走廊
  • 沿着激光指示器指向的走廊走到另一个房间

你打算从编号为 111 的房间开始，将这个策略重复执行 kkk 次，想知道自己会到达哪个房间。你觉得这个问题太简单了，于是进行了 qqq 次询问。每次询问是相互独立的，即激光指示器每次都会回到初始状态。

对于 100%100\%100% 的数据，2≤n≤8×1052 \leq n \leq 8 \times 10^52≤n≤8×105，1≤q≤8×1051 \leq q \leq 8 \times 10^51≤q≤8×105，1≤k≤10151 \leq k \leq 10^{15}1≤k≤1015，保证数据给出的是一棵树。

题目分析：

注意到 kkk 这么大，那么第一想法就是能不能倍增地跳。

如果能倍增地跳就必须保证我们任何时候跳过去后的状态相同，但是显然想要保证这一条是不可能的，所以考虑能不能发现一些性质。

因为我们如果进入了子树 uuu 后再出去，则 uuu 点的指向必然为其父亲节点，所以我们一定可以走到一种状态也就是除了 111 号节点之外，所有的点都指向其父亲。

这个东西就是很有循环节的感觉，因为当我们到达了这个状态之后，必然会经历一段不会变的序列然后再次到达这个状态，而循环的过程中每一条边最多会被经过两次，所以循环节就是 2n−22n - 22n−2 可以直接暴力存下来。

现在关键的问题就是如何快速跳过前面这些没有进入循环节的段。

可以发现如果定义其指向的点为其父亲的点为好的点，则我们每轮访问的都可以理解为包含点 111 的一个极大的均为好的点的连通块。

因为我们访问一轮过后原本连通块的内的点必然依旧被访问到，并且访问完成之后其依旧指向父亲，因此每一轮访问都相当于将连通块向外扩展一些点，因为每轮访问的都是一个包含 111 的连通块，所以我们每一轮访问过程中形成的序列都必然是我们循环节的子序列。

所以可以考虑对于每一个位置存下来其在循环节中出现的位置集合 posupos_uposu​，然后每一次扩展就可以相当于从后往前一点点地还原循环节上 posupos_uposu​ 位置的值，这个东西的感性理解就是你可以把子树内的访问看作黑盒，这样每次新扩展其实就是相当于把其上一次出现的位置还原。

因为我们必然是在上一轮的基础上扩展得到这一轮的结果，所以可以使用可持久化线段树维护每一轮访问过程中形成的序列是什么，这里其实维护的就是循环节的第 iii 个位置是否被还原了。

因为每一次访问至少会新扩展一个点，而一个点最多会被扩展一次，所以最多有 O(n)O(n)O(n) 轮，算上线段树复杂度为 O(nlog⁡n)O(n \log n)O(nlogn)。

对于一次询问就只需要判断是否在循环节之前，如果在的话就二分一下在这 O(n)O(n)O(n) 轮的哪一轮中，就可以转化为求这一个序列中的第 kkk 个数是多少。

对于求解循环节是什么我们可以一开始直接将所有的点的状态定义为指向父亲，然后暴力扫一遍即可。

对于扩展我们可以存下来上一轮中新扩展的点是什么，然后拿这些点暴力地向外扩展即可。
