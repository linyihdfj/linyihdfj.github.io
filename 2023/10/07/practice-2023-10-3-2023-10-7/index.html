CF442D Adam and Tree

题目描述：

树指的是一种无向连通图且满足任意两个点间不存在两条不同的简单路径；

彩色树指的是一种每条边都有一种颜色的树，并且同一种颜色的边集一定形成一条路径。

对于彩色树上的任意一个点 uuu，定义其权值 w(u)w(u)w(u) 为 uuu 至根节点（即 111 号点）的路径上所有边的颜色种类数。
彩色树会生长，初始时树中仅有根节点 111 号点，每个时刻都会有一个新节点长出来。每个时刻（包括初始时刻），你需要决定每条边的颜色，在满足彩色树的定义的前提下，使这个时刻树中节点权值的最大值最小。

1≤n≤1061 \le n \le 10^61≤n≤106

题目分析：

感觉上树剖就是一种很优的做法，但是显然不是很对。

一个观察就是对于 u,vu,vu,v，设 ppp 为 lca(u,v)lca(u,v)lca(u,v)，则将 u,vu,vu,v 之间的路径划分为一种颜色，与将 u,pu,pu,p 和 v,pv,pv,p 之间的路径分别划为不同的颜色对答案造成的影响一样，所以我们只需要考虑将祖先到儿子的链划分为一种颜色即可。

所以显然可以可以使用 dpdpdp 解决这个问题，而 dpdpdp 的决策就是将哪个儿子的链与这个节点到祖先的链接上，所以一个显然的想法就是设 dp[i]dp[i]dp[i] 表示以 iii 为根的子树的权值最大值最小是多少，注意这里的权值包含 iii 到父亲的边。

设 iii 儿子中 dpdpdp 值最大为 mx1mx_1mx1​ 次大为 mx2mx_2mx2​，决策显然就是让 mx1mx_1mx1​ 与 iii 到父亲的链接上，则转移为：

dp[i]=max⁡(mx1,mx2+1)dp[i] = \max(mx_1,mx_2+1) dp[i]=max(mx1​,mx2​+1)

注意对于 i=1i=1i=1 时因为不存在父亲边所以需要特判。

具体的维护方式就是每加入一个点，就从这个点暴力向上更新，直到更新不下去了就跳出，这样看上去复杂度是 O(n2)O(n^2)O(n2) 的。

但是不要忘记了树剖的想法，也就是说我们每个点的答案必然不会超过 O(log⁡n)O(\log n)O(logn)，所以每个点最多被更新 O(log⁡n)O(\log n)O(logn) 复杂度就是 O(nlog⁡n)O(n \log n)O(nlogn)。

而且通过这道题也可以发现，树剖就基本是一种最优的剖分方式了，因为这个 dpdpdp 最终跑出来的复杂度就是 O(nlog⁡n)O(n \log n)O(nlogn)。

ABC082D FT Robot

题目描述：

现有一个机器人在坐标轴原点处，朝向 xxx 轴正方向。给定一个命令串 sss，sss 内含有两种命令：F,TF,TF,T。FFF 表示向当前朝向前进 111步，TTT 表示向左或向右旋转 909090 度。请你回答，按照命令串 sss 执行，会不会到达 (x,y)(x,y)(x,y) 处。（xxx 与 yyy 由题目输入）

1≤∣s∣≤8⋅1031 \le |s| \le 8\cdot10^31≤∣s∣≤8⋅103

题目分析：

首先第一点就是对于一个极长的均为 FFF 的段，必然只会向一个方向走，所以可以对于每一段缩在一起即可。

注意到一点如果我们当前面向横着的一个方向，则我们下一次既可以选择向上也可以选择向下，也就是对于每一个 FFF 的段我们可以自由决定到底是向正方向还是负方向走。

所以直接按奇偶分类讨论然后按 x,yx,yx,y 坐标分别 dpdpdp 判断是否可以到达即可。

需要特判的就是对于第一个 TTT 的段因为我们一开始面对 xxx 正方向所以只能向正方向走。

ARC157E XXYX Binary Tree

题目描述：

给你一棵有根树，根为 111，保证对于任意一个点要么有两个儿子，要么没有儿子，你需要把每个点标号为 X，Y 之一，使得对于 n−1n-1n−1 个由父亲和儿子顺序组成的长度为 222 的字符串中：

  1. 有 AAA 个 XX。
  2. 有 BBB 个 XY。
  3. 有 CCC 个 YX。
  4. 不存在 YY。

保证 A+B+C=n−1A+B+C=n-1A+B+C=n−1，而你只需要判断是否有解。

多组数据

1≤∑n≤1041 \le \sum n \le 10^41≤∑n≤104

题目分析：

一个想法就是 dpdpdp 解决这个问题。

注意到一点就是不能存在 YY 也就是我们选 Y 的点是一个独立集，而我们每次选择一个 Y 因为其父亲必然选择 X 所以必然会产生一个 XY 的贡献，当然这个东西是除了根节点之外的。

所以我们只需要记录选择了几个 Y 以及有多少个非叶子节点的 Y 就可以解决掉 B,CB,CB,C 限制，而我们通过限制独立集的事情保证了 DDD 限制的完成，所以 AAA 限制也自然解决了。

所以直接做的话状态就爆炸了为 O(n3)O(n^3)O(n3) 而我们的 dpdpdp 值却只是记录了 010101 就很亏，所以一个想法就是能不能把 dpdpdp 的某个状态扔到值里，所以一个猜测就是当我们选择的 Y 数量固定时，选择非叶子节点的 Y 的数量必然为一段连续的区间，通过打表发现的确如此。

所以我们只需要记录一个最大值以及最小值就好了，直接记录可能就爆了，所以分两次 dpdpdp 分别做就好了。

具体的 dpdpdp 状态就是记 dp[i][j][0/1]dp[i][j][0/1]dp[i][j][0/1] 表示以 iii 为根的子树，选择了 jjj 个 Y，点 iii 是不是选择了 Y，的情况下最大/最小的选择非叶子节点 Y 的数量，转移就是树上背包，复杂度 O(n2)O(n^2)O(n2)。

需要注意的一点就是如果 CCC 是奇数同样也是不合法的，需要特判。

ARC067F Yakiniku Restaurants

题目描述：

有编号从 111 到 NNN 的 NNN 家烧烤店，烧烤店在一条线上按照编号顺序排序，第 iii 家烧烤店与第 i+1i + 1i+1 家烧烤店的距离是 AiA_iAi​。

你有编号从 111 到 MMM 的 MMM 张烧烤券，不管是在哪一家烧烤店都可用烧烤券来吃烧烤，在第 iii 家烧烤店用烧烤券 jjj 可以吃到一顿美味度为 Bi,jB_{i,j}Bi,j​ 的烧烤，每一张烧烤券只能使用一次，但是在一家烧烤店你可以使用任意多张烧烤券。

你想从自己选择的一家烧烤店开始（随意选择一个开始），然后不断地用未使用的烧烤券去另一家烧烤店。你最终的幸福值是所吃所有烧烤的美味度减去所走的总路程。求最大可能的最终幸福值（MMM 张券必须用完）。

2≤N≤5×1032 \leq N \leq 5 \times 10^32≤N≤5×103，1≤M≤2001 \leq M \leq 2001≤M≤200

题目分析：

假设我们确定了从 xxx 烧烤店开始，那么我们最优的一种策略显然是一直向左走直到 lll 然后从 lll 一直向右走直到 rrr，或者先向右后向左也是一样的。

但是这样显然不如直接从 lll 走到 rrr 优，所以我们其实只需要对于每一个 lll 一步步向右走即可。

直接枚举 l,rl,rl,r 然后再计算答案的复杂度就是 O(n2m)O(n^2m)O(n2m) 不能通过，考虑优化。

因为 mmm 个烧烤券相互独立所以可以分开考虑，所以下面只考虑对于一种烧烤券如何快速计算。

我们可以枚举一个位置，然后钦定这一张烧烤券在这个位置取得最大值，当我们将一个区间 [l,r][l,r][l,r] 看作一个二维平面上的点 (l,r)(l,r)(l,r) 时，可以发现以这个烧烤券为最大值的区间必然对应二维平面上的一个矩形，所以可以直接二维差分维护，具体的左右端点可以通过单调栈处理出左边/右边第一个大于这个值的位置即可。

因为我们只需要在最后查询一次，也就是说我们只需要做一次二维前缀和，所以复杂度就是 O(n2+nm)O(n^2 + nm)O(n2+nm).

ABC150F Xor Shift

题目描述：

给定两个长度为 nnn 的序列 a={a0,a1,⋯ ,an−1}a=\{a_0,a_1,\cdots,a_{n-1}\}a={a0​,a1​,⋯,an−1​} 和 b={b0,b1,⋯ ,bn−1}b=\{b_0,b_1,\cdots,b_{n-1}\}b={b0​,b1​,⋯,bn−1​}，输出所有有序数对 (k,x)(k,x)(k,x) ，满足：

  1. 0≤k<n0\leq k<n0≤k<n 且 x≥0x\geq 0x≥0。
  2. 序列 a′=ba'=ba′=b，其中 ai′=ai+k mod nxor⁡x (0≤i<n)a'_i = a_{i+k\bmod n}\operatorname{xor} x\ (0\leq i<n)ai′​=ai+kmodn​xorx (0≤i<n)，“xor⁡\operatorname{xor}xor”表示按位异或。

1≤n≤2×1051\leq n\leq 2\times 10^51≤n≤2×105，0≤ai,bi<2300\leq a_i,b_i<2^{30}0≤ai​,bi​<230。

题目分析：

注意到异或操作所以我们可以按位来考虑。

对于每一位问题就是相当于两个 010101 序列的循环匹配问题，循环同构的问题可以通过二倍长度来解决，也就是将 aaa 复制一遍接到 aaa 的后面，这样问题就转化为了两个 010101 序列的匹配问题，直接 KMP 就可以解决。

所以只需要对于不同位的 kkk 相同的数对暴力合并即可。

复杂度看上去是一个高于 O(nlog⁡2A)O(n \log^2 A)O(nlog2A) 的复杂度，但是因为合法的数对很少，所以跑得很快。

NNN 个球排在一起，每个球上有一个数 aia_iai​。接下来会进行若干轮删除。设现在还有 kkk 个球，则 ai=ka_i=kai​=k 的球会被删除。

最终可能球不会被删完，你需要求出最少修改几个球上的数后可以让球全部被删完。

同时还有 MMM 次修改，每次修改第 XiX_iXi​ 个球的数为 YiY_iYi​，你需要求出每次修改后上述问题的答案。

1 ≤ N ≤ 2000001\ \leq\ N\ \leq\ 2000001 ≤ N ≤ 200000，1 ≤ M ≤ 2000001\ \leq\ M\ \leq\ 2000001 ≤ M ≤ 200000，1≤Ai,Xi,Yi≤N1 \le A_i,X_i,Y_i \le N1≤Ai​,Xi​,Yi​≤N

题目分析：

ABC259G Grid Card Game

题目描述：

芷萱和诺丝正在玩一个有趣的游戏。

有一个 H×WH \times WH×W 的正方形网格，iii 行 jjj 列的点的权值为 ai,ja_{i,j}ai,j​。

  • 芷萱选择 HHH 行中的任意几行（可能为 000），给这些行上的点放上一张红色卡片。

  • 诺丝选择 WWW 列中的任意几列（可能为 000），给这些列上的点放上一张蓝色卡片。

他们计算本游戏分数的方式如下：

如果存在一个网格 (i,j)(i,j)(i,j) 满足 ai,j<0a_{i,j}<0ai,j​<0 且这个点上同时存在两种颜色的卡片，则游戏失败，分数为 −10100-10^{100}−10100 分，否则，分数为所有放了卡片（不管放了几张，不管放了什么颜色）的网格的权值之和。

1 ≤ H, W ≤ 1001\ \leq\ H,\ W\ \leq\ 1001 ≤ H, W ≤ 100，−109 ≤ Ai, j ≤ 109-10^9\ \leq\ A_{i,\ j}\ \leq\ 10^9−109 ≤ Ai, j​ ≤ 109

题目分析：

做法一：（猜结论）

注意到假设我们当前选择了 iii 行选择了 jjj 列，则我们的最优方案一定是在只考虑行的情况下选择 iii 行最优的前提下，选择 jjj 列最优的。

所以可以直接枚举选多少行，然后每次暴力判断应该选哪些列最优，复杂度 O(n3)O(n^3)O(n3)。

做法二：（正经做法）

这玩意有一种匹配的感觉，所以考虑网络流，其实就是每一个点第一次被匹配的时候贡献 ai,ja_{i,j}ai,j​，第二次被匹配的时候如果其为正数则贡献 000，否则贡献为 −∞-\infty−∞。

这样的话直接网络流建模问题就在于做不到如果选择了一行/一列，则这一行和这一列的所有元素都被选择，所以考虑最小割建模，因为最小割这个东西的限制很简单。

但是最小割的话我们就必须最小化什么东西，显然就是最小化未被选择的权值和，但是因为一个数选择两次权值不一样就很烦人，所以可以考虑先将答案设为元素和的二倍，这样我们不选择一行/一列的代价就是对应的行/列的元素和。

所以建图其实就是 SSS 向每一行连边，权值为这一行的元素和，每一列向 TTT 连边，权值为这一列的元素和，行和列之间连边，权值其实就是这一行和这一列同时被选择的代价，也就是如果当前元素大于 000 会被重复计算要删除，代价就是 ai,ja_{i,j}ai,j​，否则如果小于 000 则不能被选择两次，代价就是 ∞\infty∞。
